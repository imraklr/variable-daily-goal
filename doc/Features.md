# Features

## Version: 0.1.0

### Functional Requirements (FRs) (The system shall [perform some action].)

1. The system shall be able to replicate the entire day's schedule for the whole week in count.
2. The system shall be able to CRUD items in schedule with stateful cascading.
3. The system shall have a text input for daily comments, whose field shall remain open for the next half day.
4. The system shall be able to show weeks' schedule at one place.
5. The system shall display the basic information, preferrably on center top of screen, the date and weather in format:
    HH::MM DD:MM:YYYY 33°C Thu
6. The system shall have a timing feature which can be used to allocate time to tasks.
7. The system shall be able to use vertical and horizontal time structure stateful cascading:
    a. Vertical Stateful Cascading: All tasks in the day are assigned with a default of 30 minute gaps.
    b. Horizontal Stateful Cascading: All tasks with various timings on a day are statefully cascaded to the other days in that week.
8. The system shall preserve week work format until altered.
9. The system shall be able to show the weekly progress.

### Non-Functional Requirements (FRs) (The system shall achieve/maintain/support [quality metric].)

1. The system shall maintain a 24x7 uptime.
